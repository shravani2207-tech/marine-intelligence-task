## 2. Dataset Provenance Guide

### 2.1 River Network (river_network)
- **Generated by**: `national_gis_layer.py`, Phase 1
- **Method**: Hardcoded representative coordinate sequences per river,
  approximating each river's general course from source region to outflow
  region.
- **Basis for coordinates**: Approximate longitude/latitude pairs
  reflecting each river's known geographic path (e.g. Ganga from
  Uttarakhand through Uttar Pradesh, Bihar, to West Bengal). Not derived
  from a surveyed shapefile.
- **Confidence level**: Representative / prototype-grade. Suitable for
  demonstrating basin identity and approximate topology, not for precision
  navigation, engineering, or legal/administrative use.
- **Upgrade path**: Replace `river_coordinates` dict in
  `national_gis_layer.py` with authoritative LineStrings sourced from CWC
  or Survey of India river network shapefiles.

### 2.2 National Waterways (national_waterways)
- **Generated by**: `national_gis_layer.py`, Phase 1
- **Method**: Manually selected subset of 5 waterways (NW-1, NW-2, NW-3,
  NW-4, NW-16), chosen for known operational activity per IWAI.
- **Confidence level**: Representative subset only -- full NW1-111
  catalogue not covered (see Gap Analysis).
- **Upgrade path**: Source complete NW catalogue and geometry from
  iwai.nic.in.

### 2.3 Infrastructure Matrix (infrastructure_matrix)
- **Generated by**: `national_gis_layer.py`, Phase 1
- **Method**: Manually curated list of named dams, barrages, ports,
  jetties, bridges, and terminals with approximate coordinates, each tagged
  to its associated river.
- **Confidence level**: Names and general locations are real
  (e.g. Farakka Barrage, Sardar Sarovar Dam, Hirakud Dam), but exact
  coordinates are approximate, not GPS-surveyed.
- **Upgrade path**: Cross-reference against CWC dam registry and IWAI
  terminal registry for precise coordinates and additional categories
  (locks, reservoirs, wetlands).

### 2.4 Provenance Topology (provenance_topology)
- **Generated by**: `national_gis_layer.py`, Phase 2 -- programmatically
  derived, not manually authored.
- **Method**: For each river, all associated infrastructure nodes are
  projected onto the river's LineString using shapely's `.project()`
  (normalized position along the line), then ordered by position to infer
  upstream/downstream relationships.
- **Confidence level**: Derivation logic is sound and reproducible, but
  depends entirely on the accuracy of the underlying river geometry and
  infrastructure coordinates (both representative, per above). Correctness
  is only as strong as the inputs.
- **Reproducibility**: Fully deterministic -- re-running the script on the
  same inputs always produces the same topology.

### 2.5 Flood Risk Overlay (flood_risk_overlay)
- **Generated by**: `national_gis_layer.py`, Phase 2 -- manually assigned.
- **Method**: Qualitative HIGH/MEDIUM/LOW classification per river, based
  on well-documented historical flooding patterns (e.g. Ganga/Brahmaputra
  monsoon floodplains, Narmada/Kaveri dam-regulated flow).
- **Confidence level**: Directionally reasonable and consistent with
  publicly known flood history, but not derived from a hydrological model
  or CWC flood forecasting data.
- **Upgrade path**: Replace with model-derived risk scores from CWC flood
  forecasting or NRSC Bhuvan flood hazard atlas.

### 2.6 Navigation Corridors (navigation_corridors)
- **Generated by**: `national_gis_layer.py`, Phase 2 -- manually curated.
- **Method**: Selected based on known active cargo movement along
  NW-1 and NW-2, plus the Hirakud-Paradip coal corridor.
- **Confidence level**: Directionally correct (these are real, cited
  corridors) but only 3 of many possible corridors are represented.

### 2.7 Legacy Task 3 Datasets (ports, coastal_zones, waterways,
    environmental, logistics)
- **Generated by**: Prior Task 3 deliverable (Maritime Data Layer), see
  that repository's own README and REVIEW_PACKET for full provenance.
- **Sources cited**: shipmin.gov.in, sagarmala.gov.in, mmb.gov.in,
  iwai.nic.in, indiawris.gov.in, bhuvan.nrsc.gov.in, logistics.gov.in,
  protectedplanet.net, unep.org.
- **Known data quality note carried forward**: `ports.json` row 29 has
  duplicate coordinates with row 28 (flagged in that dataset's own notes
  field).
